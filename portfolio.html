Socket Programming Client

As part of an ECEN 424 assignment, my partner Bowen Mei and I implemented a UDP datagram talker/listener setup with simulated datagram dropping in java to prove the effacy of the algorithm in place to resend lost packets.
The program uses java threading in order to be able to react to interrupts or changes in the datagram flow. For example, should a sent datagram fail to send back an acknowledgement, the talker will resend the message. This can be demonstrated by using the faulty listener, which had a percentage chance to drop any given packet.

HACK CPU

As part of a CSCE 312 assignment, I used a series of logic gates built up from the singular NAND gate to a fully fledged functioning CPU, capable of handing assembly simple code and simple arithmetic operations.
The CPU can be run using a hardware simulator and fed simple assembly code. In addition to the CPU itself, assembly code was made that could be run on the CPU that would do various calculations (like the fibonacci sequence), identify when keys on the keyboard were pressed, and even display simple images on a monitor built into the hardware simulator.

Dash Ace (Platformer Video Game)

Game pitch: "Dash through the five zones of the Kingdom of Quick as the heir to the Dashing Throne while jumping through obstacles, taking down enemies, building your character with unique items, and defeating evil bosses to reclaim the throne from the evil king Kiltie, even if you have to try a thousand times."
My partner Benjamin Moder and I created a platformer game in the Unity engine. I was the project lead and the programmer for the game and my partner did the art for the game. The most unique aspect of the game was the procedurally generated levels, ensuring that no two runs of the game were identical. The game at the initial competition release featured 3 monster types, and endless run mode, and an endless arena mode.
